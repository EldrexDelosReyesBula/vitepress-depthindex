# Translation & i18n Guide

VitePress DepthIndex features a fully integrated **bilingual translation engine** that allows you to localize all UI messages, buttons, errors, settings, and feedback banners.

By default, the plugin includes:
* **English (`en`)**: The default language for all responses and UI elements.
* **Tagalog (`tl`)**: A complete Tagalog localization pack.

---

## 1. How the Translation Engine Works

The Translation Engine resolves keys using the `I18nAPI` helper:
1. **Language Preference**: Resolves the user's saved language (from browser `localStorage`). If none is set, it falls back to English (`en`).
2. **Dynamic Fallbacks**: If a translation key is missing in a customized pack (e.g. Tagalog `tl`), the engine automatically falls back to the English (`en`) equivalent so the UI never displays broken tokens or empty gaps.
3. **Reactive Re-Rendering**: When the user changes their language selection, panels receive a reactive broadcast event and immediately re-render using the new pack.

---

## 2. Registering a Custom Language Pack

You can register additional language packs (e.g., Spanish, French, or Japanese) dynamically at client runtime using the Plugin SDK.

### The Language Pack Interface

A translation pack is defined by the `TranslationPack` interface:

```typescript
export interface TranslationPack {
  code: string;           // Standard locale code (e.g. "es", "fr-FR")
  nativeName: string;     // Name in native characters (e.g. "Español")
  englishName: string;    // Name in English (e.g. "Spanish")
  author?: {
    name: string;
    url?: string;
  };
  keys: Record<string, string>; // Namespace-mapped translation strings
}
```

## 3. Translation Key Namespaces

To ensure complete UI coverage, a custom pack should provide translations for the following keys:

| Key Category | Key Token | Description / Context |
| :--- | :--- | :--- |
| **Intro / Panels** | `panel.title` | Main title for search panel |
| | `panel.subtitle` | Subtitle under search prompt |
| | `panel.viewing` | Label for active page context |
| | `panel.close` | Close button accessibility label |
| | `panel.poweredBy` | Powered-by attribution link |
| **Actions** | `action.copy` | Copy text action |
| | `action.copied` | Feedback notification for copy action |
| | `action.save` | Save button for edits |
| | `action.cancel` | Cancel button for edits |
| | `action.edit` | Edit message tooltip |
| | `action.summarize` | Page summarize option |
| | `action.discuss` | Page discuss option |
| | `action.report` | Error report action |
| | `action.dismiss` | Error banner dismiss action |
| **Feedback** | `feedback.helpful` | Upvote answer button |
| | `feedback.notHelpful` | Downvote answer button |
| | `feedback.copy` | Copy response |
| **Settings** | `settings.title` | Settings panel header |
| | `settings.language` | Language selector title |
| | `settings.searchMode` | Search method selector |
| | `settings.reset` | Restore defaults |
| | `settings.save` | Save configuration |
| | `settings.testConnection`| Verify Cloud API connection |
| | `settings.apiKeyInvalid` | Error warning for bad credentials |

## 5. Reactive UI Events

When switching languages, the settings dropdown dispatches a standard window event:

```typescript
window.dispatchEvent(new CustomEvent('depthindex:language-changed', {
  detail: { code: 'es' }
}));
```

The UI components listen for this event and trigger Vue state updates automatically. This ensures instant updates across all active chat bubbles and control views without requiring a page reload.
