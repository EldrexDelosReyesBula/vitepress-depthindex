# Translation & i18n

## Overview
VitePress DepthIndex features a comprehensive internationalization (i18n) framework. It enables documentation sites to serve multilingual users, adapting UI labels, settings panels, warning messages, and AI synthesis prompts to the user's preferred language.

## Built-in Languages (English, Tagalog)
Version `1.1.6` includes native support for two languages:
1. **English (`en`)**: Native default layout labels, instructions, and placeholder texts.
2. **Tagalog (`tl`)**: Full Tagalog translations for all UI components, buttons, settings options, and warning dialogues.

## How Translation Works

### Panel UI (Local)
The client-side `TranslationEngine` translates UI labels using matching string keys (such as `panel.input_placeholder` or `settings.title`). When a user changes their language, all UI labels are updated instantly without requiring a page refresh.

### Answer Translation (Hybrid/Cloud)
In hybrid and cloud modes, if the `translateAnswers` option is enabled, DepthIndex automatically requests the cloud model to translate answers into the user's selected language, ensuring consistency.

### AI Prompt Injection
When calling cloud models, the engine injects system instructions that direct the LLM to write in the target language:
- For English: Injects instructions to write answers in professional or chosen tone layouts in English.
- For Tagalog: Injects instructions directing the model to generate responses in grammatically correct Tagalog.

## Adding Community Translations
You can add custom translation packs using the `I18nAPI` within the `extensions` array:
```typescript
import { defineConfig } from 'vitepress';
import DepthIndex from 'vitepress-plugin-depthindex';

// Custom Spanish language pack
const spanishPack = {
  type: 'language',
  pack: {
    code: 'es',
    nativeName: 'Español',
    englishName: 'Spanish',
    direction: 'ltr',
    author: { name: 'Maria Dev' },
    translations: {
      'settings.title': 'Configuración de DepthIndex',
      'panel.input_placeholder': 'Pregunta algo sobre la documentación...',
      'panel.welcome_title': 'Hola, ¿cómo puedo ayudarte hoy?'
    }
  }
};

export default defineConfig({
  vite: {
    plugins: [
      DepthIndex({
        extensions: [spanishPack]
      })
    ]
  }
});
```

## Language Detection
By default, the client-side engine detects the user's browser language on first load:
- If the browser language matches an active pack (e.g. Tagalog), it loads that language pack automatically.
- Otherwise, it falls back to the default language configured for the site.

## Configuration

### Force Single Language
To lock the documentation UI to a specific language and disable user selector options:
```typescript
language: {
  forceLanguage: 'tl' // Forces Tagalog UI only
}
```

### Allow Language Selection
To permit users to switch languages via the settings panel:
```typescript
language: {
  allowUserChange: true // Default: true
}
```

### Available Languages
To limit the list of languages selectable by users:
```typescript
language: {
  availableLanguages: ['en', 'tl']
}
```

## RTL Support
DepthIndex supports Right-to-Left (RTL) layout rendering. If a language pack is configured with `direction: 'rtl'`, the panel layout, message bubbles, and text alignments will flip direction automatically.

## Contributing Translations
To contribute translation files for your native language:
1. Review the required translation keys inside [Contribute Translations Guide](/community/translate).
2. Create a translation pack conforming to the `LanguagePack` TypeScript interface.
3. Submit a pull request on GitHub or share the configuration with the community.
