# Case Study: From Open Source to AI Docs

Open-source projects often face a common challenge: a high volume of duplicate GitHub issues. Maintainers spend hours answering basic questions that are already documented.

In this case study, we look at how the community documentation site for **VitePress DepthIndex** deployed DepthIndex to reduce developer friction.

## The Challenge

Before deploying DepthIndex:
* 40% of new GitHub issues were classified as "duplicate questions" or "already documented".
* Users struggled to find configuration keys in a long documentation page.
* The documentation search only matched exact headings.

## The Implementation

The team integrated DepthIndex using the zero-config default options.

Within 48 hours:
* An interactive search bar was available inline in the docs.
* A floating assistant answered conceptual questions about installation and customization.

## The Results

After two weeks of deployment:
* **35% reduction in new GitHub Issues**: Developers found answers directly inside the documentation search bar before creating an issue.
* **Increased documentation engagement**: Average session duration increased as developers explored suggestions in the AI panel.

By making documentation interactive, the project saved hours of developer support time.
